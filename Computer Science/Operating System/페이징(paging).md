# 페이징(Paging)

페이징은 논리 주소 공간을 **페이지(Page)** 단위로,  
물리 주소 공간을 **프레임(Frame)** 단위로 나누어  
각 페이지를 프레임에 할당하는 방식입니다.

가상 메모리를 구현하기 위한 대표적인 기법입니다.

> 페이지(Page): 논리 메모리를 일정 크기로 나눈 단위  
> 프레임(Frame): 물리 메모리를 페이지와 동일한 크기로 나눈 단위

<br>

## 페이징을 사용하는 이유?

### 1. 가상 메모리 구현

- 물리 메모리보다 큰 프로세스를 실행할 수 있음
- 모든 데이터를 메모리에 올리지 않고 필요한 페이지만 적재

→ 즉, 제한된 메모리로 더 큰 프로그램 실행 가능

<br>

### 2. 메모리 관리 단순화

- 동일한 크기로 나누기 때문에 관리가 쉬움
- 복잡한 할당 알고리즘 불필요

<br>

### 3. 외부 단편화 해결

- 연속된 공간이 없어도 메모리 할당 가능
- 흩어진 공간에 배치 가능

##### 외부 단편화

> 총 메모리는 충분하지만 연속된 공간이 없어 할당이 불가능한 현상

<br>

### 4. 비연속 메모리 할당

- 프로세스를 여러 페이지로 나누어  
  물리 메모리의 여러 위치에 배치 가능

<br>

### 5. 주소 변환 단순화

- 페이지 번호로 프레임을 찾고 오프셋으로 위치 결정
- MMU(Memory Management Unit)에서 빠르게 주소 변환 가능

##### MMU (Memory Management Unit)

> 가상 주소를 물리 주소로 변환하는 하드웨어

<br>

### 6. 디스크 관리 효율

- 페이지 단위로 저장/로드
- I/O 효율 증가

<br>

## 페이징 특징

- 페이지와 프레임은 동일한 크기
- 물리 메모리는 연속적이지 않아도 됨
- 페이지 테이블을 통해 주소 변환 수행
- 외부 단편화 없음 / 내부 단편화 발생

##### 내부 단편화

> 페이지보다 작은 데이터도 하나의 페이지를 사용하기 때문에  
> 남는 공간이 발생하는 현상

<br>

### 장점

- 물리 메모리보다 큰 프로세스 실행 가능
- 메모리 관리 단순화
- 외부 단편화 제거
- 메모리 보호 가능

### 단점

- 내부 단편화 발생
- 페이지 테이블 관리 비용 증가
- 페이지 테이블 접근으로 인한 성능 저하

<br>

## 페이지 테이블 (Page Table)

논리 주소를 물리 주소로 매핑하기 위한 테이블입니다.

CPU는 페이지 테이블을 참조하여 가상 주소를 실제 물리 주소로 변환합니다.

<br>

<br>

<br>

### 특징

- 메인 메모리에 위치
- 각 프로세스는 별도의 페이지 테이블을 가짐
- 주소 변환 시 페이지 테이블 참조 필요
- 메모리 접근이 2번 발생 (페이지 테이블, 실제 데이터)

---

**PTBR(Page Table Base Register)**

CPU가 현재 실행 중인 프로세스의  
**페이지 테이블 위치를 가리키기 위해 사용하는 레지스터**입니다.

각 프로세스는 자신만의 페이지 테이블을 가지며,  
CPU는 어떤 페이지 테이블을 참조해야 하는지 알아야 합니다.

이때 PTBR이 해당 페이지 테이블의 시작 주소를 가리킵니다.

<img src="../Operating%20System/images/PTBR.png" width="670" height="299"/>

- 프로세스가 실행될 때  
  → 해당 프로세스의 페이지 테이블 주소가 PTBR에 설정

- CPU는 주소 변환 시  
  → PTBR을 기준으로 페이지 테이블에 접근

---

### 주소 변환 과정

<img src="../Operating%20System/images/페이징%20주소%20변환%20과정.png" width="700" height="335"/>

1. 메모리에서 페이지 테이블 조회 ← 메모리 접근 1번
2. 프레임 번호 획득 → 물리 주소 계산
3. 실제 데이터가 있는 메모리 접근 ← 메모리 접근 2번

<br>

이처럼 메모리 접근 시간이 두배로 늘어나게 되는데,  
이를 해결하기 위해 **TLB**라는 것을 사용합니다.

---

**TLB(Translation Lookaside Buffer)**

가상 주소 → 물리 주소 변환 속도를 높이기 위한 **캐시 메모리**입니다.

페이지 테이블의 일부 내용을 캐싱하여,  
**페이지 번호 → 프레임 번호 매핑 정보**를 저장합니다.

CPU 내부에 위치하여 빠르게 접근할 수 있습니다.

<br>

> TLB 히트 : CPU가 접근하려는 논리주소가 TLB에 있는 경우 (메모리 접근 한번)  
> TLB 미스: CPU가 접근하려는 논리주소가 TLB에 없는 경우 (메모리 접근 두번)

---

<br>

## 📌 핵심 정리

- 페이징은 가상 메모리를 구현하는 핵심 기법
- 페이지 단위로 메모리를 관리하여 효율성을 높임
- 페이지 테이블과 TLB를 통해 주소 변환 수행

<br>

> 외부 단편화는 해결하지만  
> 내부 단편화와 오버헤드가 발생한다
