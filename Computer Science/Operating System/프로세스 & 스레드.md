# 프로세스 & 스레드

<br>

> **프로세스** : 운영체제로부터 자원을 할당받은 실행 중인 프로그램  
> **스레드** : 프로세스 내에서 실행되는 작업 흐름 단위

<br>

## 프로세스

프로세스는 **실행 중인 프로그램**으로, 운영체제로부터 독립적인 메모리 공간을 할당받습니다.

<br>

### 특징

- 각 프로세스는 **독립된 주소 공간**을 가짐
- 다른 프로세스와 **메모리를 직접 공유할 수 없음**
- 하나의 프로세스가 종료되어도 다른 프로세스에 영향 없음

<br>

> 프로그램이 실행되면 하나의 프로세스가 생성

<br>

### 구조

![프로세스 메모리](../Operating%20System/images/process_memory.png)

<br>
<br>

| 영역  | 설명                   |
| ----- | ---------------------- |
| Code  | 실행할 프로그램 코드   |
| Data  | 전역 변수, static 변수 |
| Stack | 함수 호출, 지역 변수   |
| Heap  | 동적 할당 메모리       |

<br>

### 프로그램과 프로세스

#### 프로그램이란?

> 파일이 저장 장치에 저장되어 있지만 메모리에는 올라가 있지 않은 정적인 상태

| 구분     | 설명                           |
| -------- | ------------------------------ |
| 프로그램 | 실행되지 않은 코드 (파일 상태) |
| 프로세스 | 실행 중인 프로그램             |

<br>

> 프로그램이 메모리에 올라가면 프로세스가 됩니다.

<br>

## 스레드

스레드는 **프로세스 내 실행 흐름 단위**입니다.

<br>

### 특징

- **Stack 영역만 독립적으로 사용**
- Code / Data / Heap 영역은 **공유**
- 하나의 프로세스 내에서 병렬 처리 가능

<br>

> 하나의 프로세스는 여러 개의 스레드를 가질 수 있습니다.

<br>

### 구조

![스레드](../Operating%20System/images/thread.png)

<br>
<br>

스레드는 메모리를 공유하기 때문에, 하나의 스레드 오류가 전체 프로세스에 영향을 줄 수 있습니다.

ex) Segmentation Fault 발생 → 프로세스 전체 종료

<br>

## 프로세스 vs 스레드 (핵심 비교)

| 구분      | 프로세스 | 스레드 |
| --------- | -------- | ------ |
| 메모리    | 독립     | 공유   |
| 생성 비용 | 큼       | 작음   |
| 안정성    | 높음     | 낮음   |
| 오류 영향 | 개별     | 전체   |

> **프로세스 = 안정성, 스레드 = 성능**

<br>

## 멀티프로세스와 멀티스레드

### 멀티프로세스

> 여러 개의 프로세스를 통해 작업을 병렬 처리

<br>

**장점** : 높은 안정성 (프로세스 간 격리)

**단점** : 각각 독립된 메모리를 갖고 있어 작업량이 많을수록 오버헤드 발생, **Context Switching**으로 인한 성능 저하

<br>

#### Context Switching

> CPU가 실행 중인 작업을 다른 작업으로 전환하는 과정

- 현재 작업 상태 저장
- 다음 작업 상태 복원

> 프로세스는 주소 공간까지 교체되므로 캐시 미스, TLB 초기화 등 비용이 큼  
> 스레드는 동일 프로세스 내에서 동작하므로 상대적으로 가벼움

<br>

### 멀티스레드

> 하나의 프로세스 내에서 여러 스레드로 작업 처리

<br>

**장점** : 프로세스보다 생성 비용이 적고, 메모리를 공유하여 빠른 데이터 처리 가능

**단점** : 동기화 문제 발생 가능 (**Critical Section**)

<br>

#### Critical Section

> 여러 스레드가 공유 자원에 동시에 접근할 때 발생하는 문제

이를 해결하기 위해 Mutex, Spin Lock, Atomic 등의 동기화 기법을 사용합니다.

<br>

## 📌 핵심 정리

- 프로세스 = **안정성**
- 스레드 = **성능**
