# 가상 메모리(Virtual Memory)

컴퓨터 시스템에서 실제 물리적 메모리(RAM)의 크기에 관계없이,  
프로그램이 사용할 수 있는 메모리 공간을 확장하는 기술입니다.

물리 메모리가 부족하더라도 프로그램이 계속 실행될 수 있도록  
일부 데이터를 디스크에 저장하고 필요할 때 불러오는 방식으로 동작합니다.

<br>

## 메모리란?

- 프로그램과 실행에 필요한 데이터 및 코드를 저장하는 장치
- 크게 **주기억장치**와 **보조기억장치**로 구분됨

> 주기억장치: DRAM, 캐시(Cache), 레지스터(Register)  
> 보조기억장치: SSD, HDD 등

<br>

## 가상메모리 등장 배경

프로그램을 실행하기 위해서는 해당 프로세스의 데이터가  
물리 메모리(RAM)에 올라가 있어야 합니다.

하지만 초기 컴퓨터에서는  
실행하려는 프로그램의 크기가 RAM보다 크면 실행할 수 없었습니다.

이를 해결하기 위해 프로그램의 필요한 일부만 메모리에 적재하여 실행하며,  
나머지는 디스크에 저장해두었다가 필요할 때 불러오는 방식인 **가상 메모리가 등장했습니다.**

<br>

## 특징

- 프로그램 전체가 아닌 일부만 메모리에 적재하여 실행
- 디스크를 보조 기억장치(backing store)로 활용
- 작은 RAM과 큰 디스크를 결합하여 큰 메모리처럼 동작
- 각 프로세스는 독립적인 가상 주소 공간을 가짐 (메모리 보호)
- 가상 주소를 물리 주소로 변환하기 위해 **MMU** 사용

> MMU (Memory Management Unit)  
> CPU가 사용하는 가상 주소를 실제 물리 주소로 변환하는 하드웨어

## 가상 메모리 동작 방식 (요약)

가상 메모리는 주로 **페이징(Paging)** 기법을 통해 구현됩니다.

- 가상 메모리 → Page 단위
- 물리 메모리 → Frame 단위

필요한 페이지만 물리 메모리에 적재하여 실행합니다.

<br>

_자세한 동작 원리는 [페이징(Paging)](<../Operating%20System/페이징(paging).md>) 참고_

### 장점

#### 1. 메모리 확장

- 물리 메모리보다 큰 프로그램 실행 가능

#### 2. 메모리 보호

- 프로세스 간 메모리 접근 차단

#### 3. 효율적 메모리 사용

- 필요한 데이터만 적재하여 메모리 낭비 감소

#### 4. 멀티 프로세스 지원

- 여러 프로그램을 동시에 안정적으로 실행 가능

<br>

### 단점

#### 1. 페이지 부재(Page Fault) 비용

- 디스크 접근 발생 → 성능 저하

#### 2. 디스크 의존성

- 디스크 속도는 RAM보다 훨씬 느림

#### 3. 시스템 복잡성 증가

- 페이지 테이블, 교체 알고리즘 등 추가 관리 필요

<br>

## 📌 핵심 정리

- 가상 메모리는 **메모리를 확장하고 효율적으로 사용하기 위한 기술**
- 필요한 데이터만 메모리에 올려 성능과 자원 활용을 최적화

<br>

> 운영체제는 가상 메모리를 통해  
> **메모리 보호 + 효율적인 자원 관리 + 멀티 프로세스 실행**을 가능하게 합니다.
