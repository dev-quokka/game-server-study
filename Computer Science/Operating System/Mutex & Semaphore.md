# Mutex & Semaphore

멀티스레드 환경에서 여러 스레드가 공유 자원에 동시에 접근하면  
데이터 불일치와 같은 문제가 발생할 수 있습니다.

이러한 문제는 **임계 영역(Critical Section)**에서 발생하며,  
이를 해결하기 위해 동기화(Synchronization)가 필요합니다.

대표적인 동기화 도구로 **Mutex**와 **Semaphore**가 있습니다.

<br>

## 뮤텍스(Mutex)

<img src="../Operating%20System/images/뮤텍스.png" width="518" height="334"/>

뮤텍스(Mutex)는 공유 자원에 **한 번에 하나의 스레드만 접근**할 수 있도록 하는 동기화 도구입니다.

즉, **임계 영역**을 동시에 여러 스레드가 실행하지 못하도록 하여  
상호 배제(Mutual Exclusion) 를 보장합니다.

> **임계 영역(Critical Section)**  
> 여러 스레드가 동시에 접근하면 문제가 발생할 수 있는 공유 자원을 다루는 코드 영역

<br>

### 특징

- 동시에 하나의 스레드만 접근 가능
- **소유권(Ownership) 존재**
- lock을 획득한 스레드만 unlock 가능
- 임계 영역 보호 목적

<br>

### Mutex 동작 원리 (비유)

> 화장실(임계 영역)이 1칸이고 열쇠(Mutex)가 1개인 상황 (사람 = 쓰레드/프로세스)

<br>

**규칙**

- 화장실은 열쇠를 가진 사람만 입장 가능
- 사용 중이면 다른 사람은 대기
- 사용 후 열쇠 반납

<br>

<img src="../Operating%20System/images/뮤텍스 동작원리.png" width="503" height="371"/>

> 위 사진에서 여자가 이미 화장실을 사용 중이므로 열쇠가 없고,  
> 남자는 열쇠가 반환될 때까지 대기합니다.

<br>

## 세마포어(Semaphore)

<img src="../Operating%20System/images/세마포어.png" width="539" height="400"/>

세마포어(Semaphore)는 공유 자원에 접근할 수 있는  
**스레드/프로세스의 개수를 제어하는 동기화 도구**입니다.

하나의 정수 값을 통해 현재 접근 가능한 자원의 개수를 관리합니다.

<br>

### 특징

- 동시에 여러 스레드 접근 가능
- **소유권 없음**
- 다른 스레드가 해제 가능
- 자원 개수 기반 동기화

<br>

**참고**

> 세마포어 값이 1인 경우(Binary Semaphore),  
> 동작 방식은 Mutex와 유사하지만  
> **소유권 개념이 없다는 차이가 존재합니다.**

<br>

### Semaphore 동작 원리 (비유)

> 화장실(한정된 공유 자원)이 여러 칸 있고, 빈 칸 개수(Semaphore)를 표시하는 전광판이 있는 상황 (사람 = 쓰레드/프로세스)

<br>

**규칙**

- 빈 칸이 있으면 입장 (count 감소)
- 사용 후 나가면 count 증가
- 빈 칸이 없으면 대기

<img src="../Operating%20System/images/세마포어 동작원리.png" width="700" height="350"/>

> 모든 칸이 사용 중이면 count는 0이 되며,  
> 이후 들어오는 스레드는 자원이 반환될 때까지 대기합니다.
>
> 세마포어는 이 값을 통해 **동시에 접근 가능한 자원의 개수를 제어합니다.**

<br>

## Mutex vs Semaphore 비교

| 항목           | Mutex           | Semaphore          |
| -------------- | --------------- | ------------------ |
| 접근 가능 개수 | 1개             | 여러 개            |
| 소유권         | 있음            | 없음               |
| 해제 주체      | lock한 스레드만 | 다른 스레드도 가능 |
| 목적           | 상호 배제       | 자원 개수 관리     |
| 사용 예        | 임계 영역 보호  | 커넥션 풀, 작업 큐 |

<br>

## 언제 무엇을 사용할까?

- **Mutex**
  - 임계 영역 보호가 필요할 때
  - 하나의 스레드만 접근해야 할 때

- **Semaphore**
  - 제한된 자원을 여러 스레드가 공유할 때
  - 특정 개수만큼 동시에 처리해야 할 때

<br>

## 📌 핵심 정리

- Mutex는 **“하나만 허용 + 소유권 있음”**
- Semaphore는 **“여러 개 허용 + 소유권 없음”**

<br>

> 두 방식은 비슷해 보이지만  
> **소유권과 목적에서 중요한 차이가 존재합니다.**
