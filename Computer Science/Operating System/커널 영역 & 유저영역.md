# 커널 영역 & 유저 영역

## 커널이란?

커널(Kernel)은 운영체제의 핵심 부분으로,  
**항상 메모리에 상주하며 하드웨어와 응용 프로그램 사이를 중재하는 역할**을 합니다.

즉, 커널은

- CPU, 메모리, 디바이스 등의 자원을 관리하고
- 응용 프로그램이 하드웨어를 직접 다루지 않도록 **인터페이스 역할**을 수행합니다.

<img src="../Operating%20System/images/커널.png" width="465" height="318"/>

<br>

응용 프로그램은 하드웨어에 직접 접근할 수 없으며,  
반드시 커널을 통해서만 자원을 사용할 수 있습니다.

---

## 커널 영역과 유저 영역이란?

컴퓨터 시스템은 안전성과 안정성을 위해  
메모리를 크게 **커널 영역(Kernel Space)**과 **유저 영역(User Space)**으로 나눕니다.

<br>

### 🔹 커널 영역 (Kernel Space)

커널 영역은 운영체제가 실행되는 메모리 공간으로,  
모든 프로세스가 공통으로 사용하는 영역입니다.

이 공간에는 운영체제 코드, 디바이스 드라이버,  
그리고 CPU, 메모리, 디바이스를 관리하는 기능들이 포함됩니다.

커널 영역은 높은 권한을 가지며  
하드웨어에 직접 접근할 수 있습니다.

<br>

### 🔹 유저 영역 (User Space)

유저 영역은 각 프로세스마다 독립적으로 할당되는 메모리 공간으로,  
응용 프로그램이 실행되는 영역입니다.

이 영역에는 코드 영역, 데이터 영역, 힙, 스택 등이 포함되며  
프로그램 실행에 필요한 데이터가 저장됩니다.

유저 영역에서는 하드웨어에 직접 접근할 수 없으며,  
반드시 **System Call**을 통해 커널에 요청해야 합니다.

---

## System Call

> 유저 영역에서 커널 영역으로 요청을 전달하는 인터페이스

응용 프로그램은 파일 접근, 네트워크 통신, 메모리 할당 등의 작업을 수행할 때  
직접 하드웨어를 제어할 수 없기 때문에  
**System Call을 통해 커널에게 작업을 요청합니다.**

- 예: `read()`, `write()`, `open()`, `send()`

> 즉, System Call은  
> **유저 모드 → 커널 모드 전환의 진입 지점**입니다.

---

<br>

## 특징

### 🔹 커널 영역 특징

- 운영체제 및 드라이버 코드 실행
- 모든 하드웨어 접근 가능
- 높은 권한 (Kernel Mode)
- 시스템 전체에 영향

<br>

### 🔹 유저 영역 특징

- 일반 애플리케이션 실행
- 제한된 메모리 접근
- 하드웨어 직접 접근 불가
- System Call을 통해서만 커널 접근 가능

---

## 모드 전환 (User Mode ↔ Kernel Mode)

<img src="../Operating%20System/images/커널유저영역동작.png" width="580" height="249"/>

유저 영역에서 실행되던 프로그램이  
System Call을 호출하면 커널 모드로 전환됩니다.

이 과정에서는 다음과 같은 작업이 발생합니다:

- CPU 모드 변경 (User → Kernel)
- 레지스터 상태 저장
- 커널 코드 실행
- 작업 완료 후 다시 User Mode로 복귀

이 과정은 단순한 함수 호출보다 비용이 크며  
빈번하게 발생하면 성능 저하의 원인이 될 수 있습니다.

> 즉, **모드 전환 자체는 시스템에 부담이 되는 작업입니다.**

---

## 왜 영역을 나눌까?

> 핵심 목적은 **안정성과 보안**

- 사용자 프로그램 오류가 OS에 영향을 주지 않도록 보호
- 악의적인 접근으로부터 시스템 보호
- 잘못된 메모리 접근 방지

---

## 장단점

### 🔹 장점

- 시스템 안정성 확보
- 보안 강화
- 자원 접근 제어 가능

<br>

### 🔹 단점

- System Call 및 모드 전환 비용 발생
- 잦은 커널 진입 시 성능 저하

---

## 📌 핵심 정리

- 커널은 하드웨어와 프로그램 사이를 중재하는 핵심 시스템
- 시스템은 **User Space / Kernel Space**로 분리되어 동작
- 유저 프로그램은 **System Call을 통해서만 커널에 접근 가능**
- 모드 전환은 비용이 크기 때문에 성능에 영향을 줄 수 있음

<br>

> 커널과 유저 영역의 분리는 시스템의 안정성과 보안을 위한 핵심 구조입니다.
