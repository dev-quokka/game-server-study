### OSI 7계층이란?

네트워크 통신 과정을 7단계로 나눈 **개념적 모델**

<br>

#### 왜 계층으로 나눌까?

통신이 일어나는 과정이 단계별로 파악할 수 있기 때문입니다.

흐름을 한눈에 알아보기 쉽고, 사람들이 이해하기 쉽고,

7단계 중 특정한 곳에 이상이 생기면 다른 단계의 장비 및 소프트웨어를 건들이지 않고도 이상이 생긴 단계만 고칠 수 있기 때문입니다.

<br>

**예시**

```
PC방에서 게임을 하다 연결이 끊김
→ 문제 진단

1) 모든 PC가 문제
- 광랜을 제공하는 회사의 회선 문제 → 1계층 물리 계층
- 라우터의 문제 → 3계층 네트워크 계층

2) 한 PC만 문제
- 소프트웨어에 문제가 없고, 스위치에 문제 → 2계층 데이터링크 계층
- 게임 소프트웨어에 문제가 있다면 → 7계층 어플리케이션 계층
```

<br>

##### 1) 물리(Physical)

> 리피터, 케이블, 허브 등

- 데이터를 전기적인 신호로 변환해서 주고받는 기능을 진행하는 공간
- 통신 단위는 비트이며 이것은 1과 0으로 나타냄
- 단지 데이터를 송수신 할뿐 데이터가 무엇인지, 어떤 에러가 있는지 등에는 전혀 신경 쓰지 않음

<br>

##### 2) 데이터 링크(Data Link)

> 브릿지, 스위치 등

- 물리계층을 통해 송수신되는 정보의 오류와 흐름을 관리하여 안전한 정보의 전달을 수행
- Mac 주소를 통해 통신. 프레임에 Mac 주소를 부여하고 **에러검출, 재전송, 흐름제어**를 진행

<br>

##### 3) 네트워크(Network)

> 라우터, IP

- 데이터를 목적지까지 가장 안전하고 빠르게 전달하는 기능을 담당
- 라우터를 통해 이동할 경로를 선택하여 IP 주소를 지정하고, 해당 경로에 따라 패킷을 전달
- 라우팅, 흐름 제어, 오류 제어, 세그먼테이션 등을 수행

<br>

##### 4) 전송(Transport)

> TCP, UDP

- 프로세스 간 통신을 담당하며, 신뢰성 여부를 결정하는 계층
- TCP : 신뢰성, 연결지향적
- UDP : 비신뢰성, 비연결성, 실시간

```
게임 서버 관점
- TCP : 로그인, 채팅, 아이템 처리 (신뢰성 필요)
- UDP : 캐릭터 이동, 실시간 위치 동기화 (속도 중요)
```

<br>

##### 5) 세션(Session)

> API, Socket

- 데이터가 통신하기 위한 논리적 연결을 담당
- TCP/IP 세션을 만들고 없애는 책임을 지님
- 세션 설정, 유지, 종료, 전송 중단시 복구 등의 기능

<br>

##### 6) 표현(Presentation)

> JPEG, MPEG 등

- 데이터 표현에 대한 독립성을 제공하고 암호화하는 역할을 담당
- 파일 인코딩, 명령어를 포장, 압축, 암호화

<br>

##### 7) 응용(Application)

> HTTP, FTP, DNS 등

- 최종 목적지로, 응용 프로세스와 직접 관계하여 일반적인 응용 서비스를 수행
- 사용자 인터페이스, 전자우편, 데이터베이스 관리 등의 서비스를 제공

<br>

---

<br>

### 📌 핵심 정리

- OSI 7계층은 네트워크 문제를 **계층별로 나누어 이해하고 해결하기 위한 구조**
- 특히 전송 계층(TCP/UDP)은 게임 서버에서 성능과 직결되는 핵심 계층
