# 트랜잭션 (Transaction)

## 트랜잭션이란?

데이터베이스의 상태를 변경하는 작업을  
**하나의 단위로 묶어 처리하는 것**입니다.

즉, 여러 SQL 작업을 하나로 묶어  
**모두 성공하거나 모두 실패하도록 보장하는 단위**입니다.

<br>

#### 예시

상황: 아이템 구매

- UPDATE (금액 감소)
- INSERT (인벤토리 아이템 추가)
- SELECT (감소된 금액 조회)

→ 모든 작업은 하나의 트랜잭션으로 처리되어야 함

> 이처럼 여러 SQL문이 하나의 트랜잭션으로 구성될 수 있습니다.

<br>

## 트랜잭션 상태 흐름 (Commit & Rollback)

<img src="../Database/images/트랜잭션 과정.png" width="665" height="255"/>

- Active: 실행 중
- Partially Committed: 마지막 연산 완료 (Commit 직전)
- Committed: 정상 완료
- Failed: 오류 발생
- Aborted: Rollback 수행 후 상태

<br>

## Commit

> 트랜잭션이 정상적으로 완료되었을 때,  
> 모든 변경 사항을 데이터베이스에 반영하는 연산

- 트랜잭션이 성공적으로 완료되었음을 의미
- 변경 사항이 DB에 **영구 반영**

<br>

## Rollback

> 트랜잭션 실행 중 오류가 발생했을 때,  
> 모든 변경 사항을 취소하고 이전 상태로 되돌리는 연산

- 트랜잭션 실패 시 이전 상태로 복구
- 모든 변경 내용 취소 (Undo)
- 데이터 무결성 유지

<br>

## 트랜잭션의 성질 (ACID)

### Atomicity (원자성)

- 트랜잭션의 모든 작업은 전부 성공하거나 전부 실패
- 일부만 반영되는 상황은 허용되지 않음

<br>

### Consistency (일관성)

- 트랜잭션 전후 데이터는 정해진 제약 조건을 항상 만족
  > PK, FK, NOT NULL, 타입 등의 제약 유지

<br>

### Isolation (격리성)

- 동시에 실행되는 트랜잭션 간 서로의 작업에 영향을 주지 않음

> 이를 위해 데이터베이스는 **DB Lock**과 같은 메커니즘을 사용하여  
> 동시에 같은 데이터에 접근하는 것을 제어합니다.

<br>

### Durability (영속성)

- Commit된 데이터는 영구 저장

---

## DB Lock

동시에 여러 트랜잭션이 같은 데이터를 수정할 경우  
데이터 충돌이 발생할 수 있습니다.

이를 방지하기 위해  
트랜잭션 수행 중에는 해당 데이터에 Lock을 걸어  
다른 트랜잭션의 접근을 제한합니다.

---

## 📌 핵심 정리

- 트랜잭션은 작업을 하나의 단위로 묶어 처리하는 방식
- Commit → 반영 / Rollback → 취소
- ACID 특성을 통해 데이터 정합성 보장

<br>

> 트랜잭션은 데이터의 정합성과 안정성을 보장하는 핵심 메커니즘입니다.
