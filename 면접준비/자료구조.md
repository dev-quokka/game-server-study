## Array / Vector / List 질문

### vector와 list의 차이가 무엇인가요?

```
vector는 연속된 메모리 공간을 사용하기 때문에
인덱스를 통한 접근이 O(1)로 빠르고
cache locality가 좋아 실제 성능이 뛰어납니다.

반면 list는 비연속적인 메모리 구조를 사용하여
임의 접근이 불가능하고 순차 탐색이 필요해 O(n)의 비용이 발생하며,
cache 효율이 떨어집니다.

삽입/삭제의 경우
list는 위치를 알고 있을 경우에 포인터 변경만으로 O(1)에 수행할 수 있지만,
vector는 중간 삽입 시 데이터 이동이 필요해 O(n)이 소요됩니다.

또한 vector는 capacity를 초과할 경우
메모리를 재할당하고 기존 데이터를 복사하는 비용이 발생할 수 있습니다.

이러한 이유로 대부분의 경우에는 cache 효율이 좋은 vector를 사용하고,
중간 삽입/삭제가 매우 빈번한 특정 경우에는 list 사용을 고려해볼 수 있습니다.
```

<br>

### vector와 array의 차이는 무엇인가요?

```
vector와 array는 모두 연속된 메모리 공간을 사용하기 때문에
인덱스를 통한 접근이 O(1)로 빠르다는 공통점이 있습니다.

하지만 array는 생성 시 크기가 고정되며 이후 크기 변경이 불가능하지만
vector는 동적 배열로 내부적으로 capacity를 관리하며 필요 시 자동으로 크기를 확장할 수 있습니다.

이러한 특성으로 크기가 고정되고 최대 성능이 중요한 경우에는 array를,
크기가 유동적으로 변하는 경우에는 vector를 사용하는 것이 적절합니다.
```

<br>

### array, vector, list 각각 언제 사용하는 것이 좋을까요?

```
array는 프로그램 종료까지 크기가 고정되어 있고
추가적인 오버헤드 없이 가장 빠른 접근이 필요한 경우에 사용합니다.

vector는 연속된 메모리를 사용하여 접근이 빠르고
동적으로 크기를 조절할 수 있기 때문에
일반적인 상황에서 기본적으로 선택되는 컨테이너입니다.

list는 중간 삽입/삭제가 많은 경우에 유리하지만
비연속 메모리 구조로 인해 cache 효율이 낮아
실제로는 대부분 vector를 사용하는 것이 더 좋은 성능을 보입니다.

따라서 특별한 이유가 없다면 vector를 우선적으로 고려하고,
상황에 따라 array나 list를 선택합니다.
```

### 꼬리질문 대비

- #### vector에서 중간 삽입은 왜 O(n)인가요?

> 중간에 값을 삽입하려면 해당 위치 뒤의 요소들을 모두 한 칸씩 이동시켜야 합니다.  
> 최악의 경우 맨 앞에 삽입하면 모든 요소를 이동해야 하므로 O(n)이 됩니다.

<br>

- #### list에서 삽입이 O(1)인데 왜 실제로는 느릴 수 있나요?

> 삽입 자체는 O(1)이지만, 삽입할 위치를 찾기 위해 순차 탐색이 필요할 수 있어 O(n)이 추가됩니다.  
> 또한 비연속 메모리 구조로 인해 캐시 미스가 발생하여 실제 성능이 떨어질 수 있습니다.

<br>

- #### vector의 push_back은 항상 O(1)인가요?

> 일반적으로는 O(1)이 맞습니다. 하지만 값을 추가할때 capacity를 초과한다면  
> 메모리 재할당과 데이터 복사가 발생하여 O(n)이 됩니다. 따라서 평균적으로는 O(1)이 맞습니다.

<br>

- #### list는 왜 cache 효율이 떨어지나요?

> list는 노드가 메모리에 비연속적으로 존재하여  
> 다음 노드를 따라가는 과정에서 캐시 미스가 자주 발생하기 때문입니다.

<br>

- #### cache locality란 무엇인가요?

> cache locality는 CPU가 메모리에 접근할 때  
> 최근에 사용한 데이터나 인접한 데이터를 다시 사용할 가능성이 높은 특성을 의미합니다.  
> 연속된 메모리를 사용하는 vector는 cache locality가 높습니다.

<br>

- #### size와 capacity의 차이는 무엇인가요?

> size는 현재 vector에 저장된 요소의 개수이고,  
> capacity는 재할당 없이 저장할 수 있는 최대 요소 개수입니다.

<br>

- #### vector에서 메모리 재할당은 언제 발생하나요?

> size가 capacity를 초과할 때 더 큰 메모리를 할당하고  
> 기존 데이터를 복사하는 재할당이 발생합니다.  
> 이때 capacity는 2배수 비율로 증가합니다.

<br>

- #### reserve와 resize의 차이는 무엇인가요?

> reserve는 capacity만 증가시키고 size는 변하지 않습니다.  
> resize는 size를 변경하며, 새로운 요소는 특정값으로 초기화됩니다.
