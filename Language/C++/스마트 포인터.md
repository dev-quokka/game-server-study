# 스마트 포인터 (Smart Pointer)

## 개요

스마트 포인터는 C++에서 제공하는 포인터를 래핑한 클래스로,
**RAII(Resource Acquisition Is Initialization)** 원칙을 기반으로 동작합니다.

스마트 포인터를 사용하면 `delete`를 직접 호출하지 않아도 되므로  
메모리 누수(Memory Leak)나 이중 해제(Double Free) 같은 문제를 방지할 수 있습니다.

> **RAII란?**  
> 자원의 획득과 해제를 객체의 생명주기에 묶는 프로그래밍 패턴입니다.  
> 객체가 스코프를 벗어나 소멸될 때 소멸자가 자동으로 호출되어 자원을 해제합니다.

C++11부터 `<memory>` 헤더에 포함되었으며, 종류는 세 가지입니다.

| 종류         | 소유권    | 참조 카운팅 | 주요 용도                      |
| ------------ | --------- | ----------- | ------------------------------ |
| `unique_ptr` | 단독 소유 | 없음        | 단일 소유자가 명확한 경우      |
| `shared_ptr` | 공유 소유 | 있음        | 여러 곳에서 공유가 필요한 경우 |
| `weak_ptr`   | 소유 없음 | 증가 안 함  | 순환 참조 방지, 임시 접근      |

<br>

## unique_ptr

### 개념

`unique_ptr`은 가리키는 객체에 대한 **단독 소유권**을 보장하는 스마트 포인터입니다.

복사 생성자와 복사 대입 연산자가 `delete`로 막혀 있어 복사가 불가능하며,  
**`std::move()`를 통한 소유권 이전만** 가능합니다.

스코프를 벗어나는 순간 가리키는 객체의 메모리를 자동으로 해제합니다.

### 주요 메서드

| 메서드       | 설명                                                                       |
| ------------ | -------------------------------------------------------------------------- |
| `get()`      | 원시 포인터(raw pointer)를 반환, 소유권은 유지                             |
| `release()`  | 원시 포인터를 반환하고 소유권을 포기 (해제 책임이 호출자에게 넘어감)       |
| `reset(ptr)` | 기존 객체를 해제하고 새로운 포인터로 교체 (인자 없으면 `nullptr`로 초기화) |

### 예시 코드

```cpp
#include <iostream>
#include <memory>
using namespace std;

int main() {
    unique_ptr<int> u1 = make_unique<int>(10);

    // 복사 불가 - 컴파일 에러
    // unique_ptr<int> u2 = u1;

    // move를 통한 소유권 이전
    unique_ptr<int> u2 = move(u1);

    if (u1 == nullptr) cout << "u1 소유권 없음" << '\n'; // 출력됨
    cout << "u2: " << *u2 << '\n'; // 10

    // reset으로 교체
    u2.reset(new int(99));
    cout << "u2 after reset: " << *u2 << '\n'; // 99

    return 0;
}
```

### 언제 사용하나?

- 소유권이 한 곳에만 있어야 하는 경우 (가장 기본적인 선택)
- 팩토리 함수의 반환 타입
- 원시 포인터 대신 기본으로 사용하는 것을 권장

## shared_ptr

### 개념

`shared_ptr`은 여러 포인터가 **하나의 객체를 공동 소유**할 수 있는 스마트 포인터입니다.  
내부적으로 **참조 카운트(Reference Count)** 를 관리하며, 객체를 가리키는 `shared_ptr`의 수를 추적합니다.

- 새로운 `shared_ptr`이 같은 객체를 가리키면 참조 카운트 **증가**
- `shared_ptr`이 소멸되거나 다른 객체를 가리키면 참조 카운트 **감소**
- 참조 카운트가 **0이 되는 순간** 가리키는 객체의 메모리를 자동 해제

### 주요 메서드

| 메서드        | 설명                      |
| ------------- | ------------------------- |
| `use_count()` | 현재 참조 카운트 반환     |
| `get()`       | 원시 포인터 반환          |
| `reset()`     | 소유권 해제 (카운트 감소) |

### 예시 코드

```cpp
#include <iostream>
#include <memory>
using namespace std;

void Test(shared_ptr<int> s) {
    cout << "함수 내부 use_count: " << s.use_count() << '\n'; // 3
}

int main() {
    shared_ptr<int> s1 = make_shared<int>(10);
    cout << "초기 use_count: " << s1.use_count() << '\n'; // 1

    shared_ptr<int> s2 = s1; // 복사 가능, 카운트 증가
    cout << "s2 생성 후 use_count: " << s1.use_count() << '\n'; // 2

    Test(s1); // 함수 인자로 복사 전달 시 카운트 증가
    cout << "함수 반환 후 use_count: " << s1.use_count() << '\n'; // 2 (s2 여전히 존재)

    return 0;
}
```

### 주의사항

`shared_ptr`은 참조 카운팅을 위한 **제어 블록(Control Block)** 을 힙에 별도로 할당합니다.  
`make_shared<T>()`를 사용하면 객체와 제어 블록을 **한 번의 할당**으로 처리하므로 성능상 유리합니다.

```cpp
// 권장: 한 번의 힙 할당
shared_ptr<int> s1 = make_shared<int>(10);

// 비권장: 두 번의 힙 할당 (객체 + 제어 블록 별도)
shared_ptr<int> s2(new int(10));
```

<br>

### 순환 참조 (Circular Reference)

`shared_ptr` 사용 시 두 객체가 서로를 `shared_ptr`로 가리키는 **순환 참조** 상황이 발생할 수 있습니다.  
서로가 서로의 참조 카운트를 붙잡고 있어 카운트가 0이 되지 못하고, 결국 **메모리 누수**가 발생합니다.

#### 순환 참조 예시

```cpp
#include <iostream>
#include <memory>
using namespace std;

class ST {
public:
    ST()  { cout << "객체 생성" << '\n'; }
    ~ST() { cout << "객체 해제" << '\n'; }

    void Set(shared_ptr<ST> tar_) { tar = tar_; }

    shared_ptr<ST> tar; // 순환 참조 원인
};

int main() {
    shared_ptr<ST> s1 = make_shared<ST>();
    shared_ptr<ST> s2 = make_shared<ST>();

    s1->Set(s2); // s2 카운트: 2
    s2->Set(s1); // s1 카운트: 2

    cout << "s1 use_count: " << s1.use_count() << '\n'; // 2
    cout << "s2 use_count: " << s2.use_count() << '\n'; // 2

    // main 종료 시 s1, s2 소멸 → 카운트 1씩 감소 → 카운트 1로 남음
    // "객체 해제" 출력되지 않음 → 메모리 누수!
    return 0;
}
```

#### 출력 결과

```
객체 생성
객체 생성
s1 use_count: 2
s2 use_count: 2
```

소멸자가 호출되지 않아 **메모리가 해제되지 않습니다.**

## weak_ptr

### 개념

`weak_ptr`은 `shared_ptr`이 관리하는 객체를 **참조 카운트를 증가시키지 않고** 약하게 참조하는 포인터입니다.  
객체의 생명주기에 영향을 주지 않기 때문에 순환 참조 문제를 해결하는 데 사용됩니다.

`weak_ptr`은 객체를 **소유하지 않으므로** `->` 연산자를 통한 직접 접근이 불가능합니다.  
접근하려면 반드시 **`lock()`** 을 통해 `shared_ptr`로 변환해야 합니다.

### 주요 메서드

| 메서드        | 설명                                                                  |
| ------------- | --------------------------------------------------------------------- |
| `lock()`      | `shared_ptr`로 변환하여 반환. 객체가 이미 소멸되었으면 `nullptr` 반환 |
| `expired()`   | 가리키는 객체가 소멸되었으면 `true` 반환                              |
| `use_count()` | 해당 객체를 가리키는 `shared_ptr`의 수 반환                           |

### 순환 참조 해결 예시

```cpp
#include <iostream>
#include <memory>
using namespace std;

class ST {
public:
    ST()  { cout << "객체 생성" << '\n'; }
    ~ST() { cout << "객체 해제" << '\n'; }

    void Set(shared_ptr<ST> tar_) { tar = tar_; }

    weak_ptr<ST> tar; // shared_ptr → weak_ptr 변경으로 순환 참조 해소
};

int main() {
    shared_ptr<ST> s1 = make_shared<ST>();
    shared_ptr<ST> s2 = make_shared<ST>();

    s1->Set(s2); // s2 카운트: 여전히 1 (weak_ptr은 카운트 증가 안 함)
    s2->Set(s1); // s1 카운트: 여전히 1

    cout << "s1 use_count: " << s1.use_count() << '\n'; // 1
    cout << "s2 use_count: " << s2.use_count() << '\n'; // 1

    // weak_ptr을 통한 접근 예시
    if (auto locked = s1->tar.lock()) {
        cout << "s2에 정상 접근 가능" << '\n';
    }

    return 0; // 카운트 0 → 정상 소멸
}
```

#### 출력 결과

```
객체 생성
객체 생성
s1 use_count: 1
s2 use_count: 1
s2에 정상 접근 가능
객체 해제
객체 해제
```

`shared_ptr`의 참조 카운트가 1로 유지되므로 스코프 종료 시 정상적으로 소멸됩니다.

### expired() 활용

```cpp
shared_ptr<int> s = make_shared<int>(42);
weak_ptr<int> w = s;

cout << w.expired() << '\n'; // 0 (false, 아직 살아있음)

s.reset(); // shared_ptr 해제 → 카운트 0 → 객체 소멸

cout << w.expired() << '\n'; // 1 (true, 이미 소멸됨)

if (auto locked = w.lock()) {
    // 접근 불가 - lock()이 nullptr 반환
} else {
    cout << "객체가 이미 소멸됨" << '\n';
}
```

### 📌 핵심 정리

```
소유권이 하나인가?
    ├── Yes → unique_ptr
    └── No → 여러 곳에서 공유가 필요한가?
                ├── Yes → shared_ptr
                └── 소유하지 않고 참조만? → weak_ptr
```

- **원시 포인터(`new`/`delete`)는 가급적 사용하지 말기**
- `unique_ptr`을 기본으로 사용하고, 공유가 필요할 때만 `shared_ptr`로 전환
- `shared_ptr`을 사용하는 구조에서 순환 참조 가능성이 있으면 `weak_ptr` 사용
