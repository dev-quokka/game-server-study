# C++ 난수와 셔플

컴퓨터는 명령어와 알고리즘을 기반으로 동작하는 **결정론적인 기계**입니다.  
이런 특성 때문에 컴퓨터가 완전히 무작위적인 값을 자체적으로 생성하는 것은  
기본적으로 어렵습니다.

C++에서 난수를 생성하는 주요 방법은 다음과 같습니다.

- C 스타일의 rand() 함수 (지양)
- C++11에서 도입된 <random> 라이브러리 (권장)
- 하드웨어 기반 std::random_device

<br>

이 글에서는 각 방법의 특징과 주의할 점, 그리고 배열을 섞을 때 쓰는  
std::shuffle과 Fisher-Yates 알고리즘까지 함께 정리합니다.

<br>

## C 스타일 rand() - 지양하는 이유

rand() 함수는 0부터 RAND_MAX까지의 정수를 반환합니다.  
RAND_MAX는 구현마다 다르며, 표준상 최소 32767 이상이 보장됩니다.  
(Windows: 32767, Linux/macOS: 2147483647이 일반적)

<br>

특정 범위의 난수를 뽑으려면 보통 modulo 연산을 사용합니다.

```cpp
int r = rand() % 100;  // 0 ~ 99 사이
```

<br>

## rand() 문제점

rand()에는 아래와 같은 문제가 있어, 현대 C++에서는 사용을 지양합니다.

<br>

### 1. Modulo Bias (분포 편향)

rand() % N을 사용할 때, (RAND_MAX + 1)이 N으로 나누어떨어지지 않으면  
**특정 숫자가 더 자주 나오는 편향**이 발생합니다.  
이를 Modulo Bias라고 합니다.

#### 문제점

> 게임의 확률 시스템이나 시뮬레이션처럼 **공정한 분포가 중요한 곳**에서는  
> 이 편향이 누적되어 의도한 확률과 실제 결과가 달라지는 문제가 생깁니다.

<br>

#### 예시

rand()가 0~9를 반환한다고 가정하고 rand() % 3을 하면

| 결과 | 입력값     | 빈도 |
| ---- | ---------- | ---- |
| 0    | 0, 3, 6, 9 | 4번  |
| 1    | 1, 4, 7    | 3번  |
| 2    | 2, 5, 8    | 3번  |

→ 0이 한번 더 나옴

> 실제 `RAND_MAX = 32767`에서 `rand() % 1000`을 하면  
> 앞쪽 0~767이 뒤쪽보다 약 3% 더 자주 선택됩니다.

<br>

### 2. 시드 미설정 시 매 실행마다 같은 수열

rand()는 **의사난수(pseudo-random)** 생성기로, 내부 상태를 기반으로  
다음 수를 결정적으로 계산합니다. 이때 시드를 설정하지 않으면  
**기본 시드 1**로 초기화되어, 프로그램을 재실행할 때마다  
**완전히 같은 수열**이 반환됩니다.

<br>

> **의사난수(Pseudo-random)**  
> 정해진 알고리즘으로 무작위처럼 보이는 수열을 생성.  
> 시드(seed)가 같으면 항상 같은 수열이 나옴. 빠르고 재현 가능.

<br>

#### 예시

```cpp
for (int i = 0; i < 5; ++i)
    std::cout << rand() % 100 << " ";
```

> 무한히 실행해도 같은 결과 출력됩

> srand(time(nullptr))로 시드를 바꿀 수 있지만, time()은 초 단위라  
> **1초 내에 재실행하면 여전히 같은 결과**가 나옵니다.

→ 유저마다 다른 결과가 필요한 시스템에 부적합

<br>

### 3. RAND_MAX의 한계

rand()의 반환 범위는 [0, RAND_MAX]로 제한되며,  
RAND_MAX는 구현에 따라 다릅니다.

| 플랫폼  | RAND_MAX   |
| ------- | ---------- |
| Windows | 32767      |
| Linux   | 2147483647 |
| macOS   | 2147483647 |

<br>

#### 예시

Windows 환경에서는 rand()만으로 0~10만 같은 넓은 범위의 난수를  
직접 생성할 수 없습니다.

범위를 늘리려면 rand()를 여러 번 호출해 비트 연산으로 합치는 등  
추가 작업이 필요합니다.

```cpp
int r = rand() % 100000;  // RAND_MAX가 32767이라 불가능
```

→ 플랫폼 이식성 문제 및 넓은 범위의 난수 생성 불가

<br>

### 4. 난수 품질

rand()의 내부 알고리즘은 대부분 단순한 방식이라  
**통계적 패턴이 쉽게 드러나는 낮은 품질의 난수**를 생성합니다.

예를 들어 하위 비트의 주기가 짧아 `rand() & 1`로 0/1을 뽑으면  
규칙적인 패턴이 나타납니다.

→ 시뮬레이션, 암호학 등 품질이 중요한 분야에 부적합

<br>

## C++ 스타일 <random> 헤더

C++11부터 도입된 <random> 헤더는 위에서 살펴본 rand()의 문제들을  
모두 해결하며, 다양한 분포와 고품질 엔진을 제공합니다.

<br>

<random> 헤더의 난수 생성은 아래 **세 가지 요소의 조합**으로 이루어집니다.

- **시드(Seed)**: 엔진을 초기화하는 값 → std::random_device
- **엔진(Engine)**: 시드를 받아 난수 수열을 생성 → std::mt19937 등
- **분포(Distribution)**: 엔진의 결과를 원하는 범위/분포로 변환 → std::uniform_int_distribution 등

```
난수 생성 순서
[random_device (초기화)] → [mt19937 엔진 (수열 생성)] → [distribution (범위 변환)] → 최종 난수
```

<br>

### 1. std::random_device

하드웨어의 엔트로피 소스(CPU 노이즈, I/O 타이밍 등)를 이용해  
**예측 불가능한 시드**를 생성하는 클래스입니다.

```cpp
std::random_device rd;
unsigned int seed = rd();  // 매번 다른 값
```

<br>

다만 random_device 자체를 난수 생성에 계속 쓰는 것은 권장되지 않습니다.  
하드웨어 엔트로피는 생성 속도가 느리고, 구현에 따라 엔트로피가  
고갈될 수도 있기 때문입니다.

> **엔진을 초기화하는 용도로만 사용**하고, 실제 난수는 엔진에서 뽑는 것이 표준 방식

<br>

### 2. std::mt19937

mt19937은 의사난수 생성 엔진으로,  
이름의 19937은 주기가 **2^19937 - 1**이라는 의미입니다.  
현대 C++에서 가장 널리 쓰이는 범용 엔진입니다.

```cpp
std::random_device rd;
std::mt19937 gen(rd());  // random_device로 시드 받아 초기화

unsigned int r = gen();  // 난수 생성
```

<br>

| 항목            | 내용                      |
| --------------- | ------------------------- |
| 주기            | 2^19937 - 1 (사실상 무한) |
| 품질            | 통계 검정 대부분 통과     |
| 속도            | 빠름                      |
| 내부 상태 크기  | 약 2.5KB                  |
| 암호학적 안전성 | 예측 가능                 |

> 64비트 버전인 std::mt19937_64도 있으며,  
>  더 큰 범위의 난수가 필요할 때 사용합니다.

→ 게임, 시뮬레이션 등 일반적인 용도에 적합. 암호학적 용도에는 부적합.

<br>

### 3. std::uniform_int_distribution

엔진이 생성한 원시 난수를 **원하는 범위로 균등하게 변환**해주는 클래스입니다.  
내부적으로 Modulo Bias를 제거합니다.

```cpp
std::uniform_int_distribution<> dist(0, 99);  // 0~99 균등 분포
int r = dist(gen);  // 엔진을 인자로 전달
```

<br>

### 4. 기본 사용 패턴

```cpp
#include <random>

int main() {
    std::random_device rd;                       // 1. 시드 생성기
    std::mt19937 gen(rd());                      // 2. 엔진 초기화
    std::uniform_int_distribution<> dist(0, 99); // 3. 분포 지정

    for (int i = 0; i < 5; ++i)
        std::cout << dist(gen) << " ";
}
```

<br>

#### 주의할 점 - 엔진은 한 번만 생성

random_device와 mt19937은 **함수 호출마다 새로 만들면 안 됩니다.**  
mt19937의 내부 상태가 2.5KB에 달해 생성 비용이 크고,  
짧은 시간에 여러 번 초기화하면 비슷한 시드로 같은 수열이 나올 수 있습니다.

```cpp
// 나쁜 예시
int rnd(int n) {
    std::random_device rd;
    std::mt19937 gen(rd());  // 매번 재생성
    std::uniform_int_distribution<> dist(0, n);
    return dist(gen);
}

// 좋은 예시
int rnd(int n) {
    static std::random_device rd;
    static std::mt19937 gen(rd());  // 프로그램 전체에서 1번만 생성
    std::uniform_int_distribution<> dist(0, n);
    return dist(gen);
}
```

→ 엔진은 전역, static, 또는 인자 전달 방식으로 재사용

<br>

## Fisher-Yates 알고리즘

Fisher-Yates는 **배열을 균등 확률로 섞는** 표준 셔플 알고리즘입니다.  
std::shuffle의 내부 구현도 이 알고리즘이며, 모든 순열(N!가지)이  
**정확히 같은 확률로 나오는 것이 수학적으로 증명**되어 있습니다.

<br>

### 알고리즘 원리

배열의 **뒤에서 앞으로** 순회하면서, 현재 위치 i에 대해  
[0, i] 범위에서 랜덤 인덱스 j를 뽑아 arr[i]와 arr[j]를 swap합니다.

```cpp
// Fisher-Yates (std::shuffle) 직접 구현
for (int i = v.size() - 1; i > 0; i--) {
	std::uniform_int_distribution<> rnd(0, i);
	int j = rnd(gen);
	std::swap(v[i], v[j]);
}
```

핵심은 **선택 범위가 매 반복마다 줄어든다**는 점입니다.  
이미 최종 위치가 결정된 뒤쪽 원소는 건드리지 않기 때문에  
각 위치마다 정확히 한 번만 swap이 일어납니다. (O(N))

<br>

하지만 실무에서는 직접 구현 대신 std::shuffle을 사용합니다.  
내부 구현이 동일하게 Fisher-Yates이지만, 표준 라이브러리 최적화가  
적용되어 있고 실수가 없습니다.

```cpp
std::shuffle(v.begin(), v.end(), gen);  // 한 줄로 끝
```

<br>
