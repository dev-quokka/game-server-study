# 포인터 & 참조

## 포인터(Pointer)란?

포인터는 **변수의 메모리 주소를 저장하는 변수**입니다.  
타입 뒤에 \*을 붙여 포인터 타입을 선언합니다.

```cpp
int a = 10;       // int형 변수
int* p = &a;      // int형 포인터 (a의 주소 저장)

cout << p;   // a의 주소값 출력
cout << *p;  // 10 출력 (역참조)
```

<br>

#### 핵심 연산자

- `&` : 주소 연산자 (변수의 주소 반환)
- `*` : 역참조 연산자 (주소에 저장된 값 접근)

<br>

#### 특징

- 초기화 없이 선언 가능
- nullptr 저장 가능
- 다른 주소로 재할당 가능
- 간접 접근 (\* 사용)
- 포인터 연산 가능 (배열, 메모리 탐색 등)

<br>

## 참조 (Rerference)란?

이미 선언된 변수를 가리키는 별칭(Alias)입니다.

새로운 메모리 공간을 할당하지 않고, 원래 변수의 메모리 위치를 직접 공유합니다.  
참조자를 통해 값을 변경하면 원본 변수의 값도 함께 변경됩니다.

타입 뒤에 `&`을 붙여 참조를 선언합니다.

> & (주소 연산자)  
> 주소 연산자 &를 사용하면 변수에 할당된 메모리 주소를 확인할 수 있습니다.

<br>

```cpp
int a = 10;
int& ref = a;  // a의 별칭

ref = 20;
cout << ref; // 20 출력
cout << a;   // 20 출력 (원본도 변경)
```

> `&`는 참조 선언과 주소 연산자 두 가지 의미로 사용됩니다.

<br>

#### 특징

- 반드시 선언과 동시에 초기화해야 함
- nullptr 사용 불가
- 한 번 바인딩되면 다른 변수로 변경 불가
- 원본 변수와 동일한 메모리 공유

<br>

## 포인터 vs 참조 비교

| 구분      | 포인터(Pointer)            | 참조(Rerference)             |
| --------- | -------------------------- | ---------------------------- |
| 정의      | 주소를 저장하는 변수       | 변수의 별칭                  |
| 초기화    | 나중에 가능                | 선언 시 반드시 필요          |
| NULL      | 가능                       | 불가능                       |
| 재할당    | 가능                       | 불가능                       |
| 접근 방식 | \* (역참조), -> (멤버접근) | . (멤버접근)                 |
| 메모리    | 별도의 메모리 사용         | 기존 변수와 동일한 메모리 사 |

<br>

## 함수 호출 방식 (Call by Value, Call by Reference)

### Call by Value (값에 의한 전달)

값을 복사해서 전달하는 방식으로 원본 값에는 영향을 주지 않습니다.

<br>

```cpp
void func(int a) {
    a = 10;
}

int main() {
    int a = 5;
    func(a);
    cout << a; // 5
}
```

> func 내부에서 복사되어 사용되기 때문에 원본은 변하지 않습니다.

<br>

### Call by Reference (참조에 의한 전달)

변수의 **별칭(alias)** 을 전달하여 원본을 직접 수정하는 방식입니다.

<br>

```cpp
void func(int& a) {
    a = 10;
}

int main() {
    int a = 5;
    func(a);
    cout << a; // 10
}
```

> 원본을 참조하도록 하여 원본 자체도 변경됩니다.

<br>

## 사용 시 주의할 점

### 댕글링 포인터 (Dangling Pointer)

포인터가 이미 해제되거나 소멸된 메모리를 가리키고 있는 상태를 말합니다.  
역참조 시 undefined behavior가 발생합니다.

```cpp
int* p;
void func() {
    int a = 10;
    p = &a;
}
```

> a는 func 스코프를 벗어나서 소멸되면  
> p에 이후 접근하면 undefined behavior (댕글링 포인터)

<br>

### 댕글링 레퍼런스 (Dangling Reference)

참조 대상이 소멸된 이후에도 참조자가 남아있는 상태를 말합니다.  
포인터와 동일하게 undefined behavior가 발생합니다.

```cpp
int& danglingRef()
{
    int a = 10;
    return a; // 지역변수의 참조를 반환 → 함수 종료 후 소멸
}

int main(){
    int& ref = danglingRef(); // 댕글링 레퍼런스
    cout << ref;              // undefined behavior
}
```

#### 해결 방법

> 지역 변수의 참조/주소를 반환하지 않기  
> 객체의 생명주기를 명확히 관리하기

<br>

## 핵심 정리

- 포인터는 **주소를 저장하는 변수**
- 참조는 **변수의 별칭**
- 참조는 더 안전하고 간결
- 포인터는 더 유연하고 강력

<br>

> 참조는 내부적으로 포인터처럼 구현될 수 있지만,  
> 언어 차원에서는 포인터와는 다른 개념입니다.
