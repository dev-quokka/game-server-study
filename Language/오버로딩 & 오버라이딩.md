# 오버로딩 & 오버라이딩

<br>

## 바인딩(Binding)이란?

함수 호출 시 실제로 실행될 코드를 연결하는 과정입니다.

즉, 어떤 함수를 호출했을 때 컴퓨터가 정확히 어떤 코드를 실행할지 결정하는 과정입니다.

<br>

### 정적 바인딩 & 동적 바인딩

- **정적 바인딩 (Static Binding)**
  - **컴파일 시점**에 함수 호출과 실행할 코드가 연결됨
  - 함수 이름과 매개변수의 타입 및 개수를 기반으로 호출할 함수가 미리 결정됨
  - 오버로딩에서 주로 사용됨

- **동적 바인딩 (Dynamic Binding)**
  - **런타임 시점**에 함수 호출과 실행할 코드가 연결됨
  - virtual 함수를 사용할 경우, 실제 객체 타입에 따라 호출될 함수가 실행 시점에 결정됨
  - 오버라이딩에서 virtual과 함께 사용됨

<br>

## 오버로딩 (Overloading)이란?

오버로딩은 함수명이 같고 매개변수의 타입이나 개수가 다른 함수를 허용하는 것입니다.

즉, 인자의 타입이나 개수만 다르면 동일한 함수명으로 여러 개의 함수를 정의할 수 있습니다.

<br>

### 예제

```cpp
void func(int i)
{
    cout << "func(int) is called." << endl;
}

// 타입 다름 → 오버로딩 O
void func(char c)
{
    cout << "func(char) is called." << endl;
}

// 개수 다름 → 오버로딩 O
void func(int i, int j)
{
    cout << "func(int, int) is called." << endl;
}

// 리턴 타입만 다름 → 오버로딩 X (컴파일 오류)
int func(int i)
{
    cout << "int func(int) is called." << endl;
}
```

<br>

### 특징

- 컴파일 타임에 결정되는 정적 바인딩
- 함수 이름은 같지만 매개변수로 구분됨
- 리턴 타입만 다른 경우는 오버로딩이 아님

<br>

## 오버라이딩 (Overriding)이란?

오버라이딩은 상속 구조에서 부모 클래스의 함수를 자식 클래스에서 재정의하는 것입니다.

즉, 동일한 함수 시그니처를 가진 함수를 자식 클래스에서 다시 구현하여  
자식 객체에 맞는 동작을 수행하도록 합니다.

<br>

### 예제

```cpp
#include <iostream>
using namespace std;

class Base
{
public:
    virtual void func()
    {
        cout << "Base" << endl;
    }
};

class Derived : public Base
{
public:
    void func() override
    {
        cout << "Derived" << endl;
    }
};

int main()
{
    Base* ptr = new Derived;
    ptr->func();  // Derived
}
```

<br>

### 특징

- 상속 관계에서만 사용 가능
- 함수 이름, 매개변수, 반환 타입이 동일해야 함
- virtual 함수를 사용할 경우 런타임에 결정되는 동적 바인딩
- 각 자식 클래스에서 개별적인 재정의 가능

<br>

## 오버로딩 vs 오버라이딩 (핵심 비교)

| 항목      | 오버로딩                  | 오버라이딩           |
| --------- | ------------------------- | -------------------- |
| 함수 이름 | 동일                      | 동일                 |
| 매개변수  | 타입 또는 개수가 다름     | 동일해야함           |
| 반환 타입 | 달라도 가능               | 동일해야함           |
| 발생 위치 | 같은 클래스               | 상속 관계            |
| 목적      | 다양한 함수 제공          | 부모 함수 재정       |
| 실행 시간 | 컴파일 타임 (정적 바인딩) | 런타임 (동적 바인딩) |

<br>

## 📌 핵심 정리

- 오버로딩: 함수의 중복 정의 (정적 바인딩)
- 오버라이딩: 함수의 재정의 (virtual 사용 시 동적 바인딩)

<br>

> 오버라이딩은 virtual 키워드가 있어야 동적 바인딩이 발생합니다.
