# Array (배열)

## Array란?

배열은 **연속된 메모리 공간**에 데이터를 순차적으로 저장하는 자료구조입니다.

각 요소는 동일한 자료형을 가지며,  
인덱스(index)를 통해 특정 위치의 데이터에 접근할 수 있습니다.

- 요소(element): 배열에 저장된 데이터
- 인덱스(index): 요소의 위치를 나타내는 값

<br>

- 데이터가 연속된 메모리 공간에 저장됨
- 인덱스를 통해 각 요소에 접근 가능

> 참고: 대부분의 언어에서 인덱스는 0부터 시작

<br>

## 시간복잡도 (Time Complexity)

| Operation | Average | Worst |
| --------- | ------- | ----- |
| Access    | O(1)    | O(1)  |
| Insert    | O(n)    | O(n)  |
| Delete    | O(n)    | O(n)  |
| Search    | O(n)    | O(n)  |

<br>

## 장점

- 인덱스를 통해 빠른 접근 가능
- 구조가 단순하여 사용이 쉬움
- 연속된 메모리 구조로 인해 **캐시 효율이 높음**

<br>

## 단점

- 크기가 고정됨 (정적 할당)
- 중간 삽입/삭제 시 비용이 큼 (O(n))
- 크기 예측이 어려움 (메모리 낭비 또는 부족)

<br>

## 사용되는 경우

- 빠른 조회가 중요한 경우
- 데이터 크기가 크게 변하지 않는 경우
- 순차적인 데이터 저장이 필요한 경우
- 다차원 데이터 처리 (예: 행렬)

<br>

## 📌 핵심 정리

- 배열은 연속된 메모리 구조를 가지는 자료구조
- 인덱스를 통해 O(1) 접근 가능
- 삽입/삭제는 비효율적 (O(n))

<br>

> 배열은 빠른 접근이 필요한 상황에서 가장 효율적인 자료구조
