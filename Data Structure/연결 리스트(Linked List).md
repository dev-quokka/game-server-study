# 연결 리스트 (Linked List)

## 연결 리스트란?

연결 리스트는 **데이터와 포인터로 구성된 노드(Node)**들이  
서로 연결(link)되어 이루어진 **선형 자료구조**입니다.

배열과 달리 **연속된 메모리 공간을 사용하지 않고**,  
각 노드가 다음 노드의 주소를 가리키는 방식으로 구성됩니다.

이로 인해 크기가 고정된 배열의 한계를 보완할 수 있는  
**동적(Dynamic) 자료구조**입니다.

<br>

## 연결 리스트 구조

<img src="../Data Structure/images/연결 리스트.png" width="640" height="123"/>

- Head: 리스트의 시작을 가리킴
- Node: 데이터 + 다음 노드를 가리키는 포인터(next)
- 마지막 노드는 null을 가리킴

<br>

## 연결 리스트 특징

- 노드들이 포인터로 연결된 구조
- 메모리에 **비연속적으로 저장**
- 특정 위치 접근 시 **순차 탐색(O(n)) 필요**
- 삽입/삭제 시 **포인터 변경만으로 처리 가능**
- 포인터 저장을 위한 추가 메모리 필요
- cache locality가 낮아 실제 성능이 떨어질 수 있음

<br>

## 연결 리스트 종류

- **단일 연결 리스트 (Singly Linked List)**
  - next 포인터만 존재

- **이중 연결 리스트 (Doubly Linked List)**
  - prev, next 포인터 모두 존재
  - 양방향 탐색 가능

- **원형 연결 리스트 (Circular Linked List)**
  - 마지막 노드가 다시 head를 가리킴

<br>

## 장점

- 크기가 가변적 (동적 할당)
- 삽입/삭제가 효율적
  - **위치를 알고 있을 경우 O(1)**
- 메모리 재할당이 필요 없음

<br>

## 단점

- 랜덤 접근 불가능 (O(n))
- 탐색 성능이 느림
- 포인터로 인한 추가 메모리 사용
- **캐시 효율이 낮아 실제 성능이 저하될 수 있음**

<br>

## 시간 복잡도 (Time Complexity)

|                            | 시간 |
| -------------------------- | ---- |
| 탐색                       | O(n) |
| 삽입/삭제 (위치 알고 있음) | O(1) |
| 삽입/삭제 (위치 모름)      | O(n) |

<br>

## 배열 vs 연결 리스트

|           | 배열 | 연결 리스트                |
| --------- | ---- | -------------------------- |
| 메모리    | 연속 | 비연속                     |
| 접근      | O(1) | O(n)                       |
| 삽입/삭제 | O(n) | O(1) (위치 알고 있을 경우) |
| 캐시 효율 | 높음 | 낮음                       |

<br>

## 📌 핵심 정리

- 연결 리스트는 비연속 메모리를 사용하는 포인터 기반 자료구조
- 특정 위치 접근은 순차 탐색 → O(n)
- 삽입/삭제는 위치를 알고 있을 경우 O(1)
- cache locality가 낮아 실제 성능이 떨어질 수 있음

<br>

> 연결 리스트는 삽입/삭제가 유리하지만  
> 실제로는 cache 효율 때문에 vector가 더 빠른 경우가 많습니다.
