# 자료구조 (Data Structure)

## 자료구조란?

자료구조는 데이터를 효율적으로 저장하고 관리하기 위한 구조입니다.

즉, 데이터를 어떻게 저장하고 접근하며 처리할 것인가를 정의하는 방법입니다.

같은 데이터라도 어떤 구조를 사용하느냐에 따라  
**성능과 처리 방식이 크게 달라집니다.**

<br>

## 왜 자료구조를 사용할까?

데이터를 단순히 저장하는 것뿐만 아니라,  
효율적으로 처리하기 위해 적절한 자료구조를 선택해야 합니다.

- 빠른 탐색이 필요한 경우
- 삽입/삭제가 빈번한 경우
- 정렬이나 우선순위 처리가 필요한 경우

> 상황에 따라 적절한 자료구조를 선택하는 것이 중요

<br>

## 자료 구조의 분류

자료구조는 크게 다음과 같이 나눌 수 있습니다.

<br>

### 🔹 선형 자료구조 (Linear Data Structure)

<br>

<br>

데이터가 순차적으로 나열된 구조입니다.

- 배열 (Array)
- 연결 리스트 (Linked List)
- 스택 (Stack)
- 큐 (Queue)

> 데이터가 일렬로 연결되어 있으며 순차적인 접근이 특징입니다.

<br>

### 🔹 비선형 자료구조 (Non-Linear Data Structure)

데이터가 계층적이거나 그래프 형태로 연결된 구조입니다.

- 트리 (Tree)
- 그래프 (Graph)

> 데이터 간 관계가 복잡하며 다양한 구조를 가질 수 있습니다.

<br>

## 자료구조 선택 기준

자료구조는 상황에 맞게 선택해야 합니다.  
대표적인 선택 기준은 아래와 같습니다

- 데이터 접근 속도 (Access)
- 삽입/삭제 성능 (Insert/Delete)
- 메모리 사용 효율
- 데이터의 구조 및 관계

> 모든 상황에 완벽한 자료구조는 없으며,  
> 상황에 따라 trade-off가 존재합니다.

<br>

## 시간복잡도 (Time Complexity)

자료구조를 선택할 때 가장 중요한 기준 중 하나는 시간복잡도입니다.  
시간복잡도는 특정 연산이 수행되는 데 걸리는 시간의 증가 정도를 나타냅니다.

- O(1) : 상수 시간
- O(log n) : 로그 시간
- O(n) : 선형 시간
- O(n²) : 제곱 시간

> 같은 작업이라도 자료구조에 따라 성능 차이가 발생합니다.

<br>

## 자료구조와 알고리즘

자료구조와 알고리즘은 밀접한 관계를 가지고 있습니다.

- 자료구조 → 데이터를 어떻게 저장할 것인가
- 알고리즘 → 데이터를 어떻게 처리할 것인가

> 좋은 자료구조 선택은 알고리즘의 성능을 결정짓는 중요한 요소입니다.

<br>

## 📌 핵심 정리

- 자료구조는 데이터를 효율적으로 저장하고 관리하는 방법
- 상황에 따라 적절한 자료구조 선택이 중요
- 시간복잡도는 자료구조 선택의 핵심 기준
- 자료구조와 알고리즘은 서로 밀접하게 연결됨
